# 启动说明

## 项目说明

这是一个纯静态网站，不需要安装依赖，也不需要打包。页面入口是 `index.html`，运行时会读取：

- `site.config.json`：网站内容、视频、图集、导航、关于我等配置
- `hero.config.json`：首页背景轮播图和首页播放视频配置

注意：`assets/` 是本地素材目录，当前已在 `.gitignore` 中忽略，不会提交到 Git。

## 本地启动

在项目根目录执行：

```bash
python3 -m http.server 4173
```

浏览器打开：

```text
http://localhost:4173
```

不要直接双击 `index.html` 打开，因为页面需要通过 `fetch` 读取 JSON 配置文件，直接打开可能被浏览器安全策略拦截。

## 素材目录

建议按下面方式放素材：

```text
assets/
  bfq.png                  默认视频封面
  images/
    fist.png               首页背景图
    featuredImage/         精选图集图片
  videos/                  视频文件
```

配置里可以使用本地相对路径，也可以使用线上 URL。

示例：

```json
{
  "image": "assets/images/1.jpg",
  "video": "assets/videos/1.MOV"
}
```

## 常用配置

首页背景轮播：修改 `hero.config.json`

```json
{
  "interval": 10000,
  "images": [
    "assets/images/fist.png",
    "assets/images/1.jpg"
  ],
  "video": "assets/videos/10.mov"
}
```

精选图集自动滚动：修改 `site.config.json` 的 `featuredImages`

```json
{
  "interval": 10000,
  "displayCount": 4,
  "images": [
    "assets/images/featuredImage/IMG_1384.jpeg"
  ]
}
```

视频列表：修改 `site.config.json` 的 `videos`

```json
{
  "title": "视频标题",
  "image": "assets/images/1.jpg",
  "video": "assets/videos/1.MOV"
}
```

如果视频没有配置 `image`，会自动使用：

```json
"videoDefaultImage": "assets/bfq.png"
```

## 发布到 GitHub

代码已经可以直接作为静态站发布。因为素材目录没有提交，如果线上也要显示图片和视频，需要：

1. 把素材上传到服务器、对象存储或 CDN
2. 把 `site.config.json` 和 `hero.config.json` 里的素材路径改成线上 URL
3. 提交并推送配置文件

## 常见问题

页面空白或提示无法读取配置：

- 确认是通过 `python3 -m http.server 4173` 启动
- 确认 `site.config.json` 和 `hero.config.json` 在项目根目录
- 确认 JSON 格式没有多余逗号

图片或视频不显示：

- 检查配置里的路径大小写是否和本地文件完全一致
- 检查素材是否放在对应目录
- 视频文件名如 `.MOV`、`.mov`、`.MP4` 大小写需要匹配

端口被占用：

可以换一个端口：

```bash
python3 -m http.server 5173
```

然后打开：

```text
http://localhost:5173
```
